## 🛠️ Troubleshooting Common Issues

❌ **Script GUI does not appear after execution**
The Football Fusion 3 Script GUI may fail to render if the executor attaches before the game fully loads. Wait until you're standing on the field and the lobby UI disappears before attaching.
**Solution:** Restart Roblox, rejoin Football Fusion 3, wait for full load, then re-attach and execute the script.

❌ **Aimbot targets the wrong player**
This usually happens when `aimbot_fov` is set too high, causing the Football Fusion 3 Script to lock onto unintended targets outside your visual cone.
**Solution:** Lower the `aimbot_fov` value to `75` or `60` in the configuration file and set `TargetPriority` to `nearest_open`.

❌ **Speed boost causes rubber-banding**
If the server detects abnormal movement speed, it may correct your position, resulting in snapping or rubber-banding behavior.
**Solution:** Reduce the `speed_multiplier` to `1.3` or lower. Enable `SprintOnly` mode so the Football Fusion 3 Script only boosts speed when actively sprinting.

❌ **Script crashes mid-game with an error**
An outdated version of the Football Fusion 3 Script may conflict with a recent game patch, causing runtime errors.
**Solution:** Download the latest version from the button above and replace your old script files entirely. Never mix files from different versions.

❌ **Anti-AFK is not working**
Some executors override or block automated input events that the Football Fusion 3 Script uses for Anti-AFK.
**Solution:** Check your executor's settings for an "allow background input" or similar toggle and make sure it is enabled.
